## Bonsai: Grow or Die — Game Deployment

### What happened
Aaron sent a zip file (`bonsai-grow-or-die_17.zip`) containing a Next.js 15 game to deploy to Vercel.

### Recurring fix required (IMPORTANT)
Every zip Aaron sends has `"use client"` on line 1 and `// @ts-nocheck` on line 2 in `BonsaiGame.tsx`.
TypeScript requires `@ts-nocheck` to be the very FIRST line — before `"use client"`.
**Always swap these two lines before building.** Build will fail with type errors otherwise.

### Deployment flow (established pattern)
1. Extract zip to `/tmp/bonsai-deploy-NN/`
2. Move `@ts-nocheck` above `"use client"` in `components/BonsaiGame.tsx`
3. `npm install && npm run build` (verify clean)
4. `git init && git remote add origin https://github.com/azimm291955/bonsai-grow-or-die.git`
5. `git push --force origin main`
6. `vercel --yes --prod` (auto-links to `aaron-c-zimmermans-projects/bonsai-game`)
7. Live at **https://bonsai-game.vercel.app**

### Android scroll bug fix (applied in v17)
**Problem:** Game wasn't scrollable on Android.
**Root causes:**
- `overscroll-behavior-y: contain` on body blocks native touch scroll on Android
- `userScalable: false` + `maximumScale: 1` in viewport can break scroll on some Android Chrome versions

**Fix applied to `globals.css`:**
- Removed `overscroll-behavior-y: contain`
- Added `-webkit-overflow-scrolling: touch`

**Fix applied to `layout.tsx`:**
- Removed `userScalable: false`
- Changed `maximumScale: 1` → `maximumScale: 5`

### Details
Built an "Alerts & Insights" component that automatically analyzes live data and surfaces notable patterns:

**9 detection rules:**
1. WoW production drops/surges (>10/20% thresholds)
2. Weekly goal attainment (above/below with deficit sizing)
3. Sick rate monitoring (>10% critical, spiking warning, <3% success)
4. 4-week rolling trend decline/improvement
5. All-time production records
6. Low flower percentage warnings (<50%)
7. Rooms overdue for harvest (based on avg cycle time)
8. Trim team staffing alerts
9. Underperforming rooms (>30% below facility avg g/plant)

**UI features:**
- Collapsible card, placed right after stat cards
- Severity-based coloring (critical red, warning amber, success green, info blue)
- Filter tabs when >3 alerts
- Metric badges on each alert
- Auto-sorted by severity

### Why It Matters
Transforms the dashboard from a data display into a decision-support tool. Instead of scanning 20+ metrics, the CFO immediately sees what needs attention. This is the kind of proactive intelligence that saves time and catches issues early.
