### Details
Built a Room Catalog page (mirrors the Strain Catalog from #15) that provides a comprehensive at-a-glance view of all grow rooms.

**Page features:**
- Facility overview stats bar (active rooms, total production, plants, avg g/plant, canopy sqft)
- Production Share horizontal bar chart (each room's % contribution)
- Efficiency Comparison bar chart (g/plant by room with facility average reference line)
- Cards view: rich room cards with sparkline production trends, sick rate, cycle time, strain count, top strain, last harvest date
- Table view: fully sortable by 8 columns (production, g/plant, g/sqft, plants, sick %, cycle time, flower %, best strain)
- Room Size & Density section: canopy sqft, plant density, lbs/sqft, g/sqft per room

**Navigation improvements:**
- "Rooms" nav item added for managers and growers (sidebar + mobile bottom nav)
- Active nav state works on both `/rooms` and `/room/[name]` detail pages
- Room detail pages now navigate back to `/rooms` (was `/dashboard`)
- Added "Other Rooms" quick navigation at bottom of room detail pages

### Why It Matters
Rooms were previously only accessible through dashboard charts or direct URLs — now there's a dedicated comparison page for resource allocation decisions. This is the room equivalent of the Strain Catalog, completing the entity-level navigation pattern (Dashboard → Rooms → Room Detail, Dashboard → Strains → Strain Detail).

## Deploy Mode — Bonsai Game

Aaron set up a deploy workflow for the Bonsai game. When he sends a zip:

1. Unzip to `/tmp/`
2. Clone `https://github.com/azimm291955/bonsai-grow-or-die` to `deploy-temp/`
3. Copy zip contents in, `git add -A`, commit, push to main
4. Clean up temp dirs
5. Confirm with changed files + link to `https://bonsai-game.vercel.app`

Vercel auto-deploys on push. No questions, no local build checks.
