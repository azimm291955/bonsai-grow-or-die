## Bonsai Grow-or-Die Game — Pause Bug Fix (5:11 PM UTC)

### Bug Reported
Aaron reported a reproducible critical bug: pausing the game overnight causes cash to be completely drained when unpausing the next morning, triggering the Vulture Capital takeover screen. Reproduced 3 times. The date display didn't advance while paused (making it look like pause worked), but cash was still wiped.

### Root Cause Identified
The game uses **wall-clock time** to calculate the game date:
```
Date.now() - gameStartRealMs
```
This delta keeps growing even while paused. The tick loop stops (so UI doesn't re-render/update the date display), but `gameStartRealMs` is a fixed timestamp that never pauses. On unpause, the first tick fires and computes months of missed overhead in one shot, draining cash to zero.

### Fix Implemented
Three files changed in `bonsai-grow-or-die-current/bonsai-grow-or-die/`:

**`lib/types.ts`** — Added `pausedAtMs: number | null` field to `GameState` interface.

**`lib/helpers.ts`** — Added `pausedAtMs: null` to `createInitialState()` return object.

**`store/useGameStore.ts`**:
1. `migrateSavedState()` — added `if (saved.pausedAtMs === undefined) saved.pausedAtMs = null;` for old saves
2. `setPaused()` — completely rewritten:
   - On pause: records `pausedAtMs = Date.now()` in game state
   - On unpause: computes `pauseDuration = now - pausedAtMs`, shifts both `gameStartRealMs` and `lastTickRealMs` forward by that duration (erasing the pause gap from the timeline), sets `pausedAtMs = null`
3. `processTick()` — added early-return guard: `if (ui.paused || prev.pausedAtMs !== null) return;`
