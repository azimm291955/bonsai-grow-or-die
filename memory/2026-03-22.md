## Yieldbook Daily Improvement Sprint (1:00 PM UTC)

- **Improvement:** Shared SheetsProvider context for instant page navigation
- **Problem:** Every page (Dashboard, Analytics, Forecast, Rooms, Strains, Harvest, Trim) independently called `useSheets()` which fetched `/api/sheets` on mount. This meant a loading spinner appeared every time you navigated between pages — even though all 7 pages use the exact same data.
- **Solution:** Created a `SheetsProvider` React context that wraps the entire app. Data is fetched once on first authenticated page load and cached in memory. All pages now read from the shared cache instantly.
- **Key details:**
  - `app/lib/SheetsProvider.tsx` — context provider with auth-awareness (only fetches when signed in)
  - `app/providers.tsx` — client wrapper combining ClerkProvider + SheetsProvider
  - `app/lib/useSheets.ts` — now just reads from context (was standalone fetch hook)
  - `app/layout.tsx` — uses Providers wrapper instead of bare ClerkProvider
  - Auto-refresh every 5 min, manual refresh still works
  - Stale-while-revalidate: background errors don't clobber cached data
  - Prevents concurrent fetches with ref guard
- **Impact:** Navigating between any pages is now instant — no loading spinner, no API re-fetch. Feels like a single-page app instead of reloading data on every route change.
